# 3. Security Group 생성

### ✅ 개념 이해

- *Security Group(SG)**은 AWS 리소스에 적용되는 **가상 방화벽**
- **인바운드(Inbound)**: 외부에서 내 서버로 들어오는 트래픽
- **아웃바운드(Outbound)**: 내 서버에서 외부로 나가는 트래픽

> ❗ 기본 상태: 모든 인바운드 트래픽 차단, 모든 아웃바운드 허용
> 
1. **AWS 콘솔** → **VPC 서비스 → 보안 그룹(SG)** 클릭
2. **보안 그룹 생성**
    - **이름 태그**: `student01-web-sg`
    - **VPC 선택**: 이전 단계에서 만든 `student01-vpc`
    - **설명**: Web server access SG
3. **인바운드 규칙 추가**:

| 유형 | 프로토콜 | 포트 범위 | 소스 (출처) |
| --- | --- | --- | --- |
| SSH | TCP | 22 | 내 IP (혹은 0.0.0.0/0)* |
| HTTP | TCP | 80 | 0.0.0.0/0 |

> * 실습 시에는 내 IP 추천, 팀 실습 또는 서버 공개 테스트 시엔 0.0.0.0/0 허용
> 
1. 아웃바운드 규칙은 기본 상태(모두 허용) 유지

### 💡 실습 중 자주 나오는 실수

- SG를 만든 후 EC2 또는 ECS에 **적용하지 않음**
- 잘못된 포트 허용 (예: 443만 열고 80을 빼먹는 등)
- VPC 선택을 다른 것으로 하는 실수

오류 해결 방법

설명을 다음처럼 **단순하고 명확하게 영어 또는 안전한 문자만 사용**해보세요:

- `Allow SSH and HTTP access`
- `Web access for EC2`
- `Enable public SSH`
