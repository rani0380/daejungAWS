# 2. 라우팅 테이블

1. **AWS 콘솔 접속** → **VPC 서비스 → 라우팅 테이블** 클릭
2. 목록에서 생성한 라우팅 테이블 선택 (예: `student01-public-rtb`)
3. 좌측 탭 메뉴에서 **[서브넷 연결]** 클릭
4. **[서브넷 연결 편집]** 또는 **[연결 추가]** 클릭
5. 아래와 같이 서브넷 선택:

| 체크 | 서브넷 이름 | CIDR 블록 | AZ |
| --- | --- | --- | --- |
| ✅ | `student01-public-subnet-1` | `10.0.1.0/24` | ap-northeast-2a |
| ✅ | `student01-public-subnet-2` | `10.0.2.0/24` | ap-northeast-2c |
1. 저장 클릭
- **라우팅 테이블 1개에 여러 서브넷 연결 가능**
- 라우팅 테이블을 수정해도 **이미 연결된 서브넷에 바로 적용됨**
- 퍼블릭 서브넷은 반드시 **IGW로 가는 라우팅 경로(0.0.0.0/0)**와 함께 있어야 진짜 퍼블릭 기능 수행

## 🔍 한 가지 **마지막으로 꼭 확인할 것**:

👉 **라우팅 테이블 안에 IGW(인터넷 게이트웨이)로 나가는 경로가 있는가?**

### 📌 라우팅 테이블 `Ss02-rt1` 확인 방법

1. VPC → **라우팅 테이블** → `Ss02-rt1` 클릭
2. 탭: **라우팅**
3. 아래 항목이 반드시 있어야 함:

```
대상        | 대상지
------------|---------------------------
0.0.0.0/0   | igw-xxxxxxxxxxxxxxxxxxxxx
```

> 이게 없으면 퍼블릭 IP가 있어도 인터넷 연결이 되지 않습니다.
