# 4. EC2 인스턴스 생성

### ✅ 개념 이해

- *EC2(Elastic Compute Cloud)**는 AWS의 가상 컴퓨터로, 우리가 사용하는 **서버**와 같음
- 실습에서는 EC2를 통해 **SSH 접속**, **Docker 설치**, **웹 앱 실행** 등을 수행

- **EC2 콘솔 접속** → **인스턴스 시작**
- **AMI(Amazon Machine Image) 선택**
    - `Amazon Linux 2` (무료 티어 포함)
- **인스턴스 유형**
    - `t2.micro` 또는 `t3.micro` (무료 티어 or 수업 환경에 따라)
- **키 페어 생성**
    - 이름: `student01-key`
    - 형식: `.pem` (Windows는 `.ppk` 가능)
    - 반드시 다운로드 받아 보관 (재발급 불가)
- **네트워크 설정**
    - VPC: `student01-vpc`
    - 서브넷: `student01-public-subnet-1` 또는 2
    - 퍼블릭 IP 자동 할당: **활성화(Enable)**
- **보안 그룹 선택**
    - 기존 보안 그룹 선택 → `student01-web-sg`
- **인스턴스 생성 완료 후** → 연결 탭에서 **퍼블릭 IP 확인**

### 💻 SSH 접속 테스트

- 터미널 또는 Git Bash에서 다음 명령 실행 (예시):

```bash
chmod 400 student01-key.pem
ssh -i "student01-key.pem" ec2-user@<퍼블릭 IP>
접속 성공 시: Amazon Linux 2의 프롬프트가 표시됨
```

### 🔍 실습 중 자주 나오는 문제와 해결

| 문제 | 원인 | 해결 방법 |
| --- | --- | --- |
| SSH 접속 안 됨 | 보안 그룹에 22포트 없음 | SG 규칙 확인 |
| 인터넷 안 됨 | 퍼블릭 IP 없음 or 라우팅 미설정 | IGW 연결 여부 및 서브넷 확인 |
| 키 파일 오류 | 권한 문제 or 경로 오류 | `chmod 400`, 파일 위치 확인 |
