# 🏆 [제1과제] WorldPay 유저 관리 시스템 구축

---

## 1. 과제 개요

당신은 WorldPay의 유저 관리 시스템을 구축하는 업무를 수행해야 합니다.
WorldPay는 전 세계 사람들의 결제를 돕는 서비스로, **중요 정보 유출 방지와 고가용성(High Availability)** 확보가 설계의 핵심입니다.
주어진 요구사항에 따라 **보안과 확장성을 고려한 인프라**를 구축하십시오.

---

## 2. 아키텍처 구성 요소

* **컴퓨팅 / 네트워크**: VPC, EC2, ALB
* **데이터베이스**: RDS (MySQL)
* **보안 / 관리**: Secrets Manager, KMS, CloudWatch
* **소프트웨어 스택**: Python, MySQL

---

## 3. 세부 구축 요구사항

### A. 네트워크 (VPC) 및 CIDR 설계

* 리전: **ap-northeast-2 (서울)**
* 가용 영역: **AZ-A, AZ-C (2개 AZ 구성)**

### 📌 네트워크 구성

| 구분            | 리소스      | CIDR         | Name Tag           | 설명                  |
| ------------- | -------- | ------------ | ------------------ | ------------------- |
| VPC           | VPC      | 10.0.0.0/16  | WorldPay-VPC       | 기본 네트워크             |
| Public        | Subnet 1 | 10.0.1.0/24  | WorldPay-PUB-SUB-A | AZ-A / ALB, Bastion |
| Public        | Subnet 2 | 10.0.2.0/24  | WorldPay-PUB-SUB-C | AZ-C / ALB          |
| Private (App) | Subnet 1 | 10.0.11.0/24 | WorldPay-APP-SUB-A | AZ-A / EC2          |
| Private (App) | Subnet 2 | 10.0.12.0/24 | WorldPay-APP-SUB-C | AZ-C / EC2          |
| Private (DB)  | Subnet 1 | 10.0.21.0/24 | WorldPay-DB-SUB-A  | AZ-A / RDS          |
| Private (DB)  | Subnet 2 | 10.0.22.0/24 | WorldPay-DB-SUB-C  | AZ-C / RDS          |

---

### B. 컴퓨팅 및 부하 분산 (EC2 & ALB)

* OS: **Amazon Linux 2023 사용**

---

### 1) Bastion Host 구성

프라이빗 EC2 접근을 위한 관문 서버

#### 핵심 설정

* Security Group: SSH (22/tcp) 허용
* Public IP 필수

#### CLI 예시

```bash
aws ec2 run-instances \\
--image-id ami-xxxxxx \\
--instance-type t3.micro \\
--subnet-id <Public-Subnet-ID> \\
--security-group-ids <Bastion-SG-ID> \\
--associate-public-ip-address \\
--tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Bastion-EC2}]'
```

---

### 2) 고가용성 컴퓨팅 (ALB + ASG)

#### ALB

* Public Subnet 2개 이상에 배치
* Target Group 구성
* Health Check 설정 (/)

#### Auto Scaling Group

* Launch Template 사용
* UserData에 Python 환경 포함
* 최소 2개 이상 유지

---

### 3) 데이터베이스 보안

#### RDS

* Private Subnet 배치
* 외부 접근 차단

#### KMS & Secrets Manager

* **KMS Key 생성 (필수)**

  * Key Alias: **alias/WorldPay-Key**
* Secrets Manager는 반드시 해당 KMS Key를 사용하여 암호화

#### EC2 권한

* IAM Role 필요
* 권한:

  * secretsmanager:GetSecretValue
  * kms:Decrypt

---

### 4) 모니터링 및 로깅 (CloudWatch)

#### Logs

* EC2 로그 수집 (awslogs)

#### Alarm

* CPU 80% 이상 시 알림
* ASG 확장 트리거 가능

---

## 📋 구성 요소 체크리스트

| 영역  | 체크 포인트                       | 서비스          |
| --- | ---------------------------- | ------------ |
| 관리성 | Bastion → Private EC2 SSH 가능 | EC2, SG      |
| 가용성 | ALB 다중 AZ 구성                 | ALB, ASG     |
| 보안성 | DB 정보 비노출 및 KMS 암호화 적용       | Secrets, KMS |
| 운영성 | 로그 및 지표 수집                   | CloudWatch   |

---

## ⚠️ 선수 유의사항

* 보안 그룹 Outbound 80/443: Any 허용 가능
* 모든 리소스는 **Name Tag 정확히 일치 필수**
* KMS Key는 반드시 **alias/WorldPay-Key**로 생성
* 이름 불일치 시 감점

---

## 🔥 핵심 요약

* **VPC 1개 + Subnet 6개 구조 필수**
* **ALB + ASG = 고가용성 핵심**
* **RDS + Secrets + KMS = 보안 핵심**
* **CloudWatch = 운영 핵심**
