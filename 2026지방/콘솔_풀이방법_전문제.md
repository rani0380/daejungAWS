# [문제 1] Shared Network Storage 구축 (10점)

> **콘솔 90%** + SSH 접속하여 마운트 명령 필요

---

## 1-1. EFS용 보안 그룹 생성

**경로:** `EC2` → 좌측 메뉴 `보안 그룹` → `보안 그룹 생성`

| 필드 | 입력값 |
|------|--------|
| 보안 그룹 이름 | `efs-sg` |
| 설명 | `EFS Mount Target Security Group` |
| VPC | 기본 VPC 선택 |

**인바운드 규칙 추가:**

| 유형 | 프로토콜 | 포트 범위 | 소스 |
|------|----------|-----------|------|
| **NFS** | TCP | **2049** | 0.0.0.0/0 |

→ `보안 그룹 생성` 클릭

> ⚠️ **NFS = TCP 2049.** 인바운드 규칙 '유형' 드롭다운에서 NFS 선택하면 포트 자동 입력됨

---

## 1-2. EFS 파일 시스템 생성

**경로:** `EFS` → `파일 시스템 생성`

> ⚠️ **'생성' 버튼이 아니라 '사용자 지정' 버튼 클릭!** 사용자 지정으로 들어가야 Mount Target을 직접 설정할 수 있음

### [1단계] 파일 시스템 설정

| 필드 | 입력값 |
|------|--------|
| 이름 | `shared-efs` |
| 스토리지 클래스 | Standard |
| 자동 백업 | 비활성화 (대회용) |
| 수명 주기 관리 | 없음 |

→ `다음` 클릭

### [2단계] 네트워크 액세스 ⭐핵심⭐

Mount Target 설정 화면:

| 가용 영역 | 서브넷 | 보안 그룹 |
|-----------|--------|-----------|
| ap-northeast-2**a** | 첫 번째 서브넷 선택 | 기본값 제거 → **efs-sg** 선택 |
| ap-northeast-2**b** | 두 번째 서브넷 선택 | 기본값 제거 → **efs-sg** 선택 |

> ⚠️ **기본 보안 그룹을 제거하고 efs-sg로 교체해야 함!** 안 바꾸면 NFS 포트가 안 열림

→ `다음` → `다음` → `생성` 클릭

### 생성 후 확인

- 파일 시스템 ID (예: `fs-0abc1234def56789`) **메모!**
- 상태가 `사용 가능`이 될 때까지 1~2분 대기

## 1-3. EC2 인스턴스 2대 생성

**경로:** `EC2` → `인스턴스` → `인스턴스 시작`

### 인스턴스 1

| 필드 | 입력값 |
|------|--------|
| 이름 | `efs-instance-1` |
| AMI | Amazon Linux 2023 AMI |
| 인스턴스 유형 | t3.micro |
| 키 페어 | 기존 키 선택 또는 새로 생성 |

**네트워크 설정 → `편집` 클릭:**

| 필드 | 입력값 |
|------|--------|
| VPC | 기본 VPC |
| 서브넷 | **첫 번째 서브넷 (AZ-a)** |
| 퍼블릭 IP 자동 할당 | **활성화** |
| 보안 그룹 | 새 보안 그룹 생성 또는 기존 것 사용 |

**보안 그룹 인바운드:**

| 유형 | 포트 | 소스 |
|------|------|------|
| SSH | 22 | 0.0.0.0/0 |

→ `인스턴스 시작`

### 인스턴스 2

동일하게 생성하되:
- 이름: `efs-instance-2`
- 서브넷: **두 번째 서브넷 (AZ-b)** ← 이것만 다름!

---

## 1-4. SSH 접속 후 EFS 마운트

각 인스턴스의 퍼블릭 IP 확인: `EC2 > 인스턴스 > 해당 인스턴스 > 퍼블릭 IPv4 주소`

```bash
# SSH 접속
ssh -i MyKey.pem ec2-user@<퍼블릭IP>
```

### 각 인스턴스에서 실행 (인스턴스1, 인스턴스2 모두):

```bash
# 1) EFS 유틸리티 설치
sudo yum install -y amazon-efs-utils

# 2) 마운트 디렉토리 생성
sudo mkdir -p /data

# 3) EFS 마운트 (fs-xxxxxxxx를 실제 EFS ID로 교체!)
sudo mount -t efs fs-xxxxxxxx:/ /data

# 4) 마운트 확인
df -h | grep efs

# 5) 영구 마운트 등록
echo "fs-xxxxxxxx:/ /data efs defaults,_netdev 0 0" | sudo tee -a /etc/fstab
```

> 💡 **EFS ID 확인:** EFS 콘솔 > 파일 시스템 > 파일 시스템 ID 열에서 복사

## 2-5. 외부 테이블 생성 ⭐핵심⭐

데이터베이스가 `sales_db`로 선택된 상태에서:

```sql
CREATE EXTERNAL TABLE sales_db.sales (
    order_id INT,
    product_name STRING,
    category STRING,
    quantity INT,
    price BIGINT
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
LOCATION 's3://athena-data-2026-xxxxx/data/'
TBLPROPERTIES ('skip.header.line.count'='1');
```

→ `실행` 클릭

### 각 부분 설명

| 구문 | 의미 |
|------|------|
| `CREATE EXTERNAL TABLE` | S3 데이터를 직접 참조하는 외부 테이블 |
| `ROW FORMAT DELIMITED` | 구분자 기반 형식 |
| `FIELDS TERMINATED BY ','` | 컬럼 구분자 = 쉼표 (CSV) |
| `LINES TERMINATED BY '\n'` | 행 구분자 = 줄바꿈 |
| `LOCATION 's3://...'` | 데이터 파일 위치 (**끝에 / 필수!**) |
| `skip.header.line.count` = `1` | 첫 줄(헤더) 건너뛰기 |

### 테이블 확인

```sql
SELECT * FROM sales_db.sales LIMIT 5;
```

→ 데이터가 정상 출력되면 성공

## 3-3. IAM 정책 생성 ⭐핵심⭐

**경로:** `IAM` → `정책` → `정책 생성`

→ **`JSON` 탭 클릭** (시각적 편집기 대신 JSON 직접 입력)

아래 JSON을 **버킷명만 실제 값으로 바꿔서** 붙여넣기:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowListBucket",
            "Effect": "Allow",
            "Action": [
                "s3:ListBucket"
            ],
            "Resource": "arn:aws:s3:::company-data-2026-xxxxx",
            "Condition": {
                "StringLike": {
                    "s3:prefix": [
                        "user-data/*"
                    ]
                }
            }
        },
        {
            "Sid": "AllowGetBucketLocation",
            "Effect": "Allow",
            "Action": [
                "s3:GetBucketLocation"
            ],
            "Resource": "arn:aws:s3:::company-data-2026-xxxxx"
        },
        {
            "Sid": "AllowObjectOperationsInPrefix",
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject",
                "s3:GetObjectVersion",
                "s3:GetObjectAcl",
                "s3:PutObjectAcl"
            ],
            "Resource": "arn:aws:s3:::company-data-2026-xxxxx/user-data/*"
        },
        {
            "Sid": "ExplicitlyDenyDeleteObject",
            "Effect": "Deny",
            "Action": [
                "s3:DeleteObject",
                "s3:DeleteObjectVersion"
            ],
            "Resource": "arn:aws:s3:::company-data-2026-xxxxx/*"
        }
    ]
}
```

→ `다음`

| 필드 | 입력값 |
|------|--------|
| 정책 이름 | `S3LimitedAccessPolicy` |
| 설명 | `Fine-grained S3 access for specific prefix` |

→ `정책 생성`

### 4개 Statement 구조 요약

| # | Sid | Effect | Action | Resource | 조건 |
|---|-----|--------|--------|----------|------|
| ① | AllowListBucket | Allow | s3:ListBucket | `:::버킷명` | prefix = user-data/* |
| ② | AllowGetBucketLocation | Allow | s3:GetBucketLocation | `:::버킷명` | 없음 |
| ③ | AllowObjectOps | Allow | Get/PutObject 등 5개 | `:::버킷명/user-data/*` | 없음 |
| ④ | **DenyDelete** | **Deny** | DeleteObject 2개 | `:::버킷명/*` | 없음 (**전체 차단**) |

### ARN 패턴 구분 (자주 틀리는 부분!)

| 대상 | ARN | 사용처 |
|------|-----|--------|
| 버킷 수준 | `arn:aws:s3:::버킷명` (콜론 3개, 뒤에 아무것도 없음) | ListBucket, GetBucketLocation |
| 특정 prefix 객체 | `arn:aws:s3:::버킷명/user-data/*` | GetObject, PutObject |
| 전체 객체 | `arn:aws:s3:::버킷명/*` | Deny 규칙 |

## 3-4. 정책을 사용자에 연결

## 4-7. Lambda 코드 입력

함수 생성 후 `코드` 탭에서 `lambda_function.py` 파일 내용을 **전부 지우고** 아래 코드 붙여넣기:

```python
import json
import pymysql
import os

def get_connection():
    return pymysql.connect(
        host=os.environ["DB_HOST"],
        user=os.environ["DB_USER"],
        password=os.environ["DB_PASS"],
        database=os.environ["DB_NAME"],
        cursorclass=pymysql.cursors.DictCursor,
        connect_timeout=10
    )

def lambda_handler(event, context):
    conn = get_connection()
    method = event.get("httpMethod", "")
    
    try:
        with conn.cursor() as cur:
            
            # POST: 데이터 입력
            if method == "POST":
                body = json.loads(event.get("body", "{}"))
                name = body.get("name", "")
                description = body.get("description", "")
                
                cur.execute(
                    "INSERT INTO items (name, description) VALUES (%s, %s)",
                    (name, description)
                )
                conn.commit()
                
                return {
                    "statusCode": 201,
                    "headers": {
                        "Content-Type": "application/json",
                        "Access-Control-Allow-Origin": "*"
                    },
                    "body": json.dumps({
                        "message": "Created successfully",
                        "id": cur.lastrowid
                    })
                }
            
            # GET: 데이터 조회
            elif method == "GET":
                cur.execute("SELECT * FROM items ORDER BY id DESC")
                rows = cur.fetchall()
                
                return {
                    "statusCode": 200,
                    "headers": {
                        "Content-Type": "application/json",
                        "Access-Control-Allow-Origin": "*"
                    },
                    "body": json.dumps(rows, default=str)
                }
            
            else:
                return {
                    "statusCode": 400,
                    "headers": {"Content-Type": "application/json"},
                    "body": json.dumps({"error": f"Unsupported method: {method}"})
                }
    
    except Exception as e:
        return {
            "statusCode": 500,
            "headers": {"Content-Type": "application/json"},
            "body": json.dumps({"error": str(e)})
        }
    
    finally:
        conn.close()
```

→ **`Deploy` 버튼 클릭!** (코드 저장)

### 코드 핵심 포인트

| 항목 | 설명 |
|------|------|
| `os.environ["DB_HOST"]` | 환경 변수에서 DB 접속 정보 읽음 |
| `DictCursor` | 결과를 딕셔너리로 반환 (JSON 변환 편리) |
| `event["httpMethod"]` | API Gateway 프록시 통합으로 전달된 HTTP 메서드 |
| `event["body"]` | POST 요청의 본문 (문자열 → json.loads 필요) |
| `json.dumps(rows, default=str)` | datetime 등을 문자열로 자동 변환 |
| `conn.close()` | finally에서 반드시 연결 종료 |

## 문제4 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `Task timed out after 3.00 seconds` | Lambda 기본 timeout | timeout 30초로 변경 |
| `No module named 'pymysql'` | Layer 미추가 | Layer 추가 확인, python/ 구조 확인 |
| `Can't connect to MySQL server` | VPC 미설정 또는 보안그룹 | Lambda VPC 설정, 3306 포트 확인 |
| `Access denied for user` | 비밀번호 오류 | 환경변수 DB_PASS 확인 |
| `Unknown database 'mydb'` | RDS에 초기 DB 안 만듦 | mysql 접속해서 `CREATE DATABASE mydb;` |
| `Internal Server Error` from API GW | Lambda 권한 없음 | 콘솔에서 저장 시 자동 부여됨 (재설정) |
| `Missing Authentication Token` | 잘못된 URL 또는 미배포 | URL 경로 확인, API 배포 확인 |
| `KeyError: 'httpMethod'` | 프록시 통합 미체크 | API GW에서 Lambda 프록시 통합 ✅ |
| POST는 되는데 GET 안됨 | GET 메서드 미생성 | API GW에서 GET 메서드 추가 |
| 배포 후에도 안됨 | 이전 배포본 캐시 | 새로 배포 (`작업` > `API 배포`) |

## 문제4 체크리스트

| ☐ | 확인 항목 |
|---|-----------|
| ☐ | 보안 그룹에 MySQL 3306 인바운드 |
| ☐ | RDS 생성 시 '추가 구성'에서 초기 DB 이름 = mydb |
| ☐ | RDS 퍼블릭 액세스 = 예 |
| ☐ | RDS 보안 그룹 = rds-lambda-sg |
| ☐ | items 테이블 생성 완료 |
| ☐ | Lambda Role에 VPCAccessExecutionRole + BasicExecutionRole |
| ☐ | pymysql Layer가 python/ 구조로 패키징 |
| ☐ | Lambda에 Layer 추가됨 |
| ☐ | Lambda 환경변수 4개 (DB_HOST, DB_USER, DB_PASS, DB_NAME) |
| ☐ | Lambda timeout = 30초 |
| ☐ | Lambda VPC 설정 (서브넷 2개 + 보안그룹) |
| ☐ | Lambda 코드 Deploy 완료 |
| ☐ | Lambda 단독 테스트 성공 (GET/POST) |
| ☐ | API Gateway = REST API (HTTP API 아님!) |
| ☐ | /items 리소스 생성됨 |
| ☐ | GET + POST 메서드 모두 생성 |
| ☐ | **Lambda 프록시 통합 ✅ 체크** |
| ☐ | **API 배포 완료 (prod 스테이지)** |
| ☐ | curl POST → 201 Created |
| ☐ | curl GET → 입력 데이터 조회됨 |

# 전체 리소스 정리 (대회 후)

## 정리 순서 (의존성 역순)

| 순서 | 서비스 | 경로 | 작업 |
|------|--------|------|------|
| 1 | API Gateway | API Gateway > ItemsAPI | API 삭제 |
| 2 | Lambda | Lambda > rds-crud-function | 함수 삭제 |
| 3 | Lambda Layer | Lambda > 계층 > pymysql-layer | 계층 삭제 |
| 4 | RDS | RDS > my-mysql | 삭제 (최종 스냅샷 건너뛰기) |
| 5 | EFS | EFS > shared-efs | 먼저 Mount Target 삭제 → EFS 삭제 |
| 6 | EC2 | EC2 > 인스턴스 | 인스턴스 종료 |
| 7 | IAM | IAM > 역할/사용자/정책 | 정책 분리 → 삭제 |
| 8 | S3 | S3 > 버킷 | 객체 비우기 → 버킷 삭제 |
| 9 | 보안 그룹 | EC2 > 보안 그룹 | 사용 중인 것 없는지 확인 후 삭제 |
