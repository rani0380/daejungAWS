
## STEP 2. 보안 그룹 생성

```bash
# RDS + Lambda 공용 보안그룹
RDS_SG=$(aws ec2 create-security-group \
    --group-name rds-lambda-sg \
    --description "RDS MySQL + Lambda SG" \
    --vpc-id $VPC_ID \
    --query "GroupId" --output text)

# MySQL 포트 허용
aws ec2 authorize-security-group-ingress \
    --group-id $RDS_SG \
    --protocol tcp \
    --port 3306 \
    --cidr 0.0.0.0/0

echo "Security Group:$RDS_SG"
```

> ⚠️ **MySQL = TCP 3306** (NFS는 2049, SSH는 22 — 헷갈리지 말 것!)
>

## STEP 3. RDS 서브넷 그룹 생성

```bash
aws rds create-db-subnet-group \
    --db-subnet-group-name my-db-subnet-group \
    --db-subnet-group-description "Subnet group for Lambda RDS" \
    --subnet-ids $SUBNET1 $SUBNET2
```

> 🧠 **왜 필요한가?** RDS는 고가용성을 위해 최소 2개 AZ의 서브넷이 필요. 서브넷 그룹 없이는 RDS 생성 불가!

## STEP 4. RDS MySQL 인스턴스 생성 ⭐시간 소요⭐

```bash
aws rds create-db-instance \
    --db-instance-identifier my-mysql \
    --db-instance-class db.t3.micro \
    --engine mysql \
    --engine-version "8.0" \
    --master-username admin \
    --master-user-password "MyPass1234!" \
    --allocated-storage 20 \
    --db-subnet-group-name my-db-subnet-group \
    --vpc-security-group-ids $RDS_SG \
    --db-name mydb \
    --no-multi-az \
    --publicly-accessible \
    --backup-retention-period 0
```

### 각 옵션 상세 설명

| 옵션 | 값 | 설명 |
| --- | --- | --- |
| `--db-instance-identifier` | my-mysql | RDS 인스턴스 이름 (고유) |
| `--db-instance-class` | db.t3.micro | 인스턴스 크기 (프리티어) |
| `--engine` | mysql | 데이터베이스 엔진 |
| `--engine-version` | “8.0” | MySQL 버전 |
| `--master-username` | admin | 마스터 사용자명 |
| `--master-user-password` | “MyPass1234!” | 마스터 비밀번호 (8자 이상, 특수문자 포함) |
| `--allocated-storage` | 20 | 스토리지 크기 (GB) |
| `--db-subnet-group-name` | my-db-subnet-group | 위에서 만든 서브넷 그룹 |
| `--vpc-security-group-ids` | $RDS_SG | 보안 그룹 ID |
| `--db-name` | mydb | **초기 데이터베이스 이름 (필수!)** |
| `--no-multi-az` | - | 단일 AZ (대회용, 비용 절약) |
| `--publicly-accessible` | - | 퍼블릭 액세스 허용 |
| `--backup-retention-period` | 0 | 백업 비활성화 (대회용, 생성 빠름) |

## STEP 6. Lambda IAM 역할 생성

### 6-1. Trust Policy 생성

```bash
cat > trust-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "lambda.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
EOF
```

### Trust Policy 구조 설명

| 필드 | 값 | 의미 |
| --- | --- | --- |
| Principal.Service | lambda.amazonaws.com | Lambda 서비스가 이 역할을 수임(Assume)할 수 있음 |
| Action | sts:AssumeRole | 역할 수임 허용 |

### 6-2. 역할 생성 및 정책 연결

```bash
# 역할 생성
ROLE_ARN=$(aws iam create-role \
    --role-name lambda-rds-role \
    --assume-role-policy-document file://trust-policy.json \
    --query "Role.Arn" --output text)

echo "Role ARN:$ROLE_ARN"

# VPC 접근 권한 연결 (Lambda가 VPC 내 RDS에 접근하려면 필수!)
aws iam attach-role-policy \
    --role-name lambda-rds-role \
    --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaVPCAccessExecutionRole

# 기본 실행 권한도 연결 (CloudWatch 로그 등)
aws iam attach-role-policy \
    --role-name lambda-rds-role \
    --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
```

### Lambda에 필요한 정책 정리

| 정책 | ARN | 용도 |
| --- | --- | --- |
| AWSLambdaVPCAccessExecutionRole | `arn:aws:iam::aws:policy/service-role/AWSLambdaVPCAccessExecutionRole` | Lambda가 VPC 내부 리소스(RDS) 접근 |
| AWSLambdaBasicExecutionRole | `arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole` | CloudWatch Logs 기록 |

> 🧠 **VPCAccessExecutionRole이 없으면?** Lambda가 VPC 내의 RDS에 접근 불가 → Connection timeout 에러
> 

> ⚠️ **역할 생성 후 10초 정도 대기** 필요 (IAM 전파 시간). 바로 Lambda 생성하면 역할 인식 안 될 수 있음.

## STEP 8. Lambda 함수 코드 작성 ⭐핵심⭐

### 8-1. 함수 코드 생성

```bash
cat > lambda_function.py << 'PYEOF'
import json
import pymysql
import os

def get_connection():
    """RDS MySQL 연결 생성"""
    return pymysql.connect(
        host=os.environ["DB_HOST"],
        user=os.environ["DB_USER"],
        password=os.environ["DB_PASS"],
        database=os.environ["DB_NAME"],
        cursorclass=pymysql.cursors.DictCursor,
        connect_timeout=10
    )

def lambda_handler(event, context):
    """메인 핸들러 - GET(조회) / POST(입력) 처리"""
    conn = get_connection()
    method = event.get("httpMethod", "")

    try:
        with conn.cursor() as cur:

            # ===== POST: 데이터 입력 =====
            if method == "POST":
                body = json.loads(event.get("body", "{}"))
                name = body.get("name", "")
                description = body.get("description", "")

                cur.execute(
                    "INSERT INTO items (name, description) VALUES (%s, %s)",
                    (name, description)
                )
                conn.commit()

                return {
                    "statusCode": 201,
                    "headers": {
                        "Content-Type": "application/json",
                        "Access-Control-Allow-Origin": "*"
                    },
                    "body": json.dumps({
                        "message": "Created successfully",
                        "id": cur.lastrowid
                    })
                }

            # ===== GET: 데이터 조회 =====
            elif method == "GET":
                cur.execute("SELECT * FROM items ORDER BY id DESC")
                rows = cur.fetchall()

                return {
                    "statusCode": 200,
                    "headers": {
                        "Content-Type": "application/json",
                        "Access-Control-Allow-Origin": "*"
                    },
                    "body": json.dumps(rows, default=str)
                }

            # ===== 기타 메서드 =====
            else:
                return {
                    "statusCode": 400,
                    "headers": {"Content-Type": "application/json"},
                    "body": json.dumps({"error": f"Unsupported method: {method}"})
                }

    except Exception as e:
        return {
            "statusCode": 500,
            "headers": {"Content-Type": "application/json"},
            "body": json.dumps({"error": str(e)})
        }

    finally:
        conn.close()
PYEOF
```

### Lambda 코드 핵심 포인트

| 항목 | 내용 |
| --- | --- |
| DB 접속 | `os.environ["DB_HOST"]` → 환경 변수에서 읽음 |
| 커서 타입 | `DictCursor` → 결과를 딕셔너리로 반환 (JSON 변환 편리) |
| POST 처리 | `event["body"]`에서 JSON 파싱 → INSERT |
| GET 처리 | SELECT 전체 → `json.dumps(rows, default=str)` |
| 응답 형식 | `{"statusCode": 200, "headers": {...}, "body": "..."}` |
| default=str | datetime 등 직렬화 불가 타입을 문자열로 변환 |
| CORS | `Access-Control-Allow-Origin: *` 헤더 추가 |
| 에러 처리 | try/except/finally로 연결 반드시 close |

### event 객체 구조 (API Gateway Lambda Proxy 통합 시)

```json
{
  "httpMethod": "POST",
  "path": "/items",
  "headers": { "Content-Type": "application/json" },
  "body": "{\"name\":\"Widget\",\"description\":\"Test\"}",
  "queryStringParameters": null,
  "pathParameters": null
}
```

> 🧠 **Lambda Proxy 통합이어야** `event["httpMethod"]`, `event["body"]` 사용 가능!
> 

### Lambda 응답 형식 (반드시 이 구조!)

```json
{
  "statusCode": 200,
  "headers": {
    "Content-Type": "application/json"
  },
  "body": "{\"key\":\"value\"}"
}
```

> ⚠️ **body는 반드시 문자열!** 딕셔너리가 아니라 `json.dumps()`로 변환한 문자열이어야 함
>

## STEP 9. Lambda 함수 배포

### 9-1. 코드 패키징 및 함수 생성

```bash
# zip 패키징
zip function.zip lambda_function.py

# Lambda 함수 생성
LAMBDA_ARN=$(aws lambda create-function \
    --function-name rds-crud-function \
    --runtime python3.12 \
    --handler lambda_function.lambda_handler \
    --role $ROLE_ARN \
    --zip-file fileb://function.zip \
    --layers $LAYER_ARN \
    --vpc-config SubnetIds=$SUBNET1,$SUBNET2,SecurityGroupIds=$RDS_SG \
    --environment "Variables={DB_HOST=$RDS_ENDPOINT,DB_USER=admin,DB_PASS=MyPass1234!,DB_NAME=mydb}" \
    --timeout 30 \
    --memory-size 256 \
    --query "FunctionArn" --output text)

echo "Lambda ARN:$LAMBDA_ARN"
```

### 각 옵션 상세 설명

| 옵션 | 값 | 설명 |
| --- | --- | --- |
| `--function-name` | rds-crud-function | 함수 이름 |
| `--runtime` | python3.12 | Python 런타임 버전 |
| `--handler` | lambda_function.lambda_handler | 파일명.함수명 |
| `--role` | $ROLE_ARN | 실행 역할 ARN |
| `--zip-file` | fileb://function.zip | 코드 패키지 |
| `--layers` | $LAYER_ARN | pymysql Layer ARN |
| `--vpc-config` | SubnetIds=…,SecurityGroupIds=… | **VPC 설정 (RDS 접근 필수!)** |
| `--environment` | Variables={DB_HOST=…} | **환경 변수 4개** |
| `--timeout` | 30 | 제한 시간 30초 (기본 3초는 너무 짧음!) |
| `--memory-size` | 256 | 메모리 256MB |

> ⚠️ **자주 실수하는 3가지:**
1. `--layers` 빠뜨림 → `import pymysql` 실패
2. `--vpc-config` 빠뜨림 → RDS 연결 timeout
3. `--timeout 30` 빠뜨림 → 3초 기본값으로 timeout
>

## STEP 10. API Gateway 생성 ⭐핵심⭐

### 10-1. REST API 생성

```bash
API_ID=$(aws apigateway create-rest-api \
    --name "ItemsAPI" \
    --description "CRUD API for items" \
    --query "id" --output text)

echo "API ID:$API_ID"
```

### 10-2. 루트 리소스 ID 가져오기

```bash
ROOT_ID=$(aws apigateway get-resources \
    --rest-api-id $API_ID \
    --query "items[0].id" --output text)

echo "Root Resource ID:$ROOT_ID"
```

### 10-3. /items 리소스 생성

```bash
RESOURCE_ID=$(aws apigateway create-resource \
    --rest-api-id $API_ID \
    --parent-id $ROOT_ID \
    --path-part "items" \
    --query "id" --output text)

echo "Resource ID:$RESOURCE_ID"
```

### API 구조

```
/                ← ROOT_ID (자동 생성)
└── /items       ← RESOURCE_ID (직접 생성)
    ├── GET      ← 데이터 조회
    └── POST     ← 데이터 입력
```

### 10-4. GET 메서드 생성 + Lambda 연결

```bash
# GET 메서드 생성
aws apigateway put-method \
    --rest-api-id $API_ID \
    --resource-id $RESOURCE_ID \
    --http-method GET \
    --authorization-type NONE

# GET → Lambda 통합 (Lambda Proxy)
aws apigateway put-integration \
    --rest-api-id $API_ID \
    --resource-id $RESOURCE_ID \
    --http-method GET \
    --type AWS_PROXY \
    --integration-http-method POST \
    --uri "arn:aws:apigateway:${REGION}:lambda:path/2015-03-31/functions/${LAMBDA_ARN}/invocations"
```

### 10-5. POST 메서드 생성 + Lambda 연결

```bash
# POST 메서드 생성
aws apigateway put-method \
    --rest-api-id $API_ID \
    --resource-id $RESOURCE_ID \
    --http-method POST \
    --authorization-type NONE

# POST → Lambda 통합 (Lambda Proxy)
aws apigateway put-integration \
    --rest-api-id $API_ID \
    --resource-id $RESOURCE_ID \
    --http-method POST \
    --type AWS_PROXY \
    --integration-http-method POST \
    --uri "arn:aws:apigateway:${REGION}:lambda:path/2015-03-31/functions/${LAMBDA_ARN}/invocations"
```

### put-integration 옵션 상세

| 옵션 | 값 | 설명 |
| --- | --- | --- |
| `--type` | **AWS_PROXY** | Lambda 프록시 통합 (event에 전체 요청 전달) |
| `--integration-http-method` | **POST** | ⚠️ **항상 POST!** GET 메서드라도 Lambda 호출은 POST |
| `--uri` | `arn:aws:apigateway:...` | Lambda 함수 호출 URI |

### URI 형식 (암기!)

```
arn:aws:apigateway:{REGION}:lambda:path/2015-03-31/functions/{LAMBDA_ARN}/invocations
```

> ⚠️ **integration-http-method는 항상 POST!** 이것은 API Gateway가 Lambda를 호출하는 방법이지, 클라이언트의 HTTP 메서드가 아님
> 

> ⚠️ **AWS_PROXY vs AWS:**
- `AWS_PROXY` = Lambda 프록시 통합 → event에 httpMethod, body 등 전체 전달 (우리가 사용할 것!)
- `AWS` = 사용자 지정 통합 → 매핑 템플릿 직접 구성 필요 (복잡)
> 

---

## STEP 11. Lambda 실행 권한 부여

```bash
# API Gateway가 Lambda를 호출할 수 있도록 권한 부여
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)

aws lambda add-permission \
    --function-name rds-crud-function \
    --statement-id apigateway-invoke-all \
    --action lambda:InvokeFunction \
    --principal apigateway.amazonaws.com \
    --source-arn "arn:aws:execute-api:${REGION}:${ACCOUNT_ID}:${API_ID}/*"
```

### add-permission 옵션 설명

| 옵션 | 값 | 설명 |
| --- | --- | --- |
| `--statement-id` | apigateway-invoke-all | 권한 식별자 (고유) |
| `--action` | lambda:InvokeFunction | Lambda 호출 허용 |
| `--principal` | apigateway.amazonaws.com | API Gateway 서비스 |
| `--source-arn` | `.../API_ID/*` | 특정 API의 모든 메서드 |

> ⚠️ **이 권한 없으면?** API Gateway에서 Lambda 호출 시 500 Internal Server Error 발생

# 트러블슈팅 가이드

---

## 자주 발생하는 에러와 해결 방법

| 에러 | 원인 | 해결 |
| --- | --- | --- |
| `Task timed out after 3.00 seconds` | Lambda 기본 timeout 3초 | timeout을 30초로 변경 |
| `No module named 'pymysql'` | Layer 미추가 또는 구조 잘못 | python/ 폴더 안에 패키지 확인, Layer 재추가 |
| `Can't connect to MySQL server (timeout)` | Lambda VPC 미설정 또는 보안그룹 | vpc-config 확인, 3306 포트 열림 확인 |
| `Access denied for user` | DB 비밀번호 오류 | 환경변수 DB_PASS 확인 |
| `Unknown database 'mydb'` | RDS 생성 시 –db-name 누락 | RDS에 접속해서 CREATE DATABASE mydb; |
| `Internal Server Error (500)` from API GW | Lambda 권한 미부여 | add-permission으로 apigateway 호출 권한 추가 |
| `Missing Authentication Token` from API GW | 잘못된 URL 또는 미배포 | create-deployment 실행, URL 경로 확인 |
| `KeyError: 'httpMethod'` | Lambda 프록시 통합 미설정 | API GW에서 AWS_PROXY 타입으로 변경 |
| `body is not JSON serializable` | response body에 dict 직접 전달 | json.dumps()로 문자열 변환 |
| `datetime is not serializable` | datetime 객체 JSON 변환 실패 | json.dumps(data, default=str) 사용 |

# 리소스 정리

```bash
# 1. API Gateway 삭제
aws apigateway delete-rest-api --rest-api-id $API_ID

# 2. Lambda 함수 삭제
aws lambda delete-function --function-name rds-crud-function

# 3. Lambda Layer 삭제
aws lambda delete-layer-version --layer-name pymysql-layer --version-number 1

# 4. Lambda 권한 제거 (함수 삭제하면 자동 삭제되므로 선택사항)

# 5. RDS 삭제 (스냅샷 없이)
aws rds delete-db-instance \
    --db-instance-identifier my-mysql \
    --skip-final-snapshot

# 6. RDS 서브넷 그룹 삭제 (RDS 삭제 완료 후)
aws rds delete-db-subnet-group --db-subnet-group-name my-db-subnet-group

# 7. IAM 정책 분리 및 역할 삭제
aws iam detach-role-policy --role-name lambda-rds-role \
    --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaVPCAccessExecutionRole
aws iam detach-role-policy --role-name lambda-rds-role \
    --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
aws iam delete-role --role-name lambda-rds-role

# 8. 보안 그룹 삭제
aws ec2 delete-security-group --group-id $RDS_SG
```

> ⚠️ **정리 순서 중요!** API GW → Lambda → Layer → RDS(대기) → 서브넷그룹 → IAM → 보안그룹
