Shared Network Storage
Amazon EFS를 생성하고 여러 EC2 인스턴스에 공유 마운트하는 시나리오
아키텍처 구성
EC2 #1
↘
EFS (NFS)
↗
EC2 #2
두 EC2가 동일한 EFS 마운트 포인트를 공유
⚠️
<변수>는 실제 값으로 바꿔야 합니다. 예: <your-vpc-id> → vpc-0abc1234
1
VPC & 서브넷 확인
bash · AWS CLI
# 기본 VPC ID 확인
aws ec2 describe-vpcs \
  --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" \
  --output text

# 서브넷 목록 확인
aws ec2 describe-subnets \
  --filters Name=vpc-id,Values=<vpc-id> \
  --query "Subnets[*].[SubnetId,AvailabilityZone]" \
  --output table
2
EFS용 Security Group 생성
bash · AWS CLI
# EFS Security Group 생성 (NFS 포트 2049 허용)
aws ec2 create-security-group \
  --group-name efs-sg \
  --description "Security group for EFS" \
  --vpc-id <vpc-id>

# NFS 인바운드 허용 (EC2 SG로부터)
aws ec2 authorize-security-group-ingress \
  --group-id <efs-sg-id> \
  --protocol tcp \
  --port 2049 \
  --source-group <ec2-sg-id>
3
EFS 파일 시스템 생성
bash · AWS CLI
# EFS 생성
aws efs create-file-system \
  --performance-mode generalPurpose \
  --throughput-mode bursting \
  --encrypted \
  --tags Key=Name,Value=shared-efs

# 생성된 EFS ID 확인
aws efs describe-file-systems \
  --query "FileSystems[?Name=='shared-efs'].FileSystemId" \
  --output text
4
Mount Target 생성 (서브넷마다)
bash · AWS CLI
# 첫 번째 서브넷에 마운트 타겟 생성
aws efs create-mount-target \
  --file-system-id <efs-id> \
  --subnet-id <subnet-id-1> \
  --security-groups <efs-sg-id>

# 두 번째 서브넷에도 생성 (멀티 AZ)
aws efs create-mount-target \
  --file-system-id <efs-id> \
  --subnet-id <subnet-id-2> \
  --security-groups <efs-sg-id>

# 마운트 타겟 상태 확인 (available 될 때까지 대기)
aws efs describe-mount-targets \
  --file-system-id <efs-id>
5
EC2에서 EFS 마운트
💡
EC2에 SSH 접속 후 아래 명령어 실행. amazon-efs-utils 패키지 필요
bash · EC2 SSH
# EFS utils 설치 (Amazon Linux 2)
sudo yum install -y amazon-efs-utils

# 마운트 디렉토리 생성
sudo mkdir -p /mnt/efs

# EFS 마운트 (TLS 암호화)
sudo mount -t efs -o tls <efs-id>:/ /mnt/efs

# 부팅 시 자동 마운트 (/etc/fstab 등록)
echo "<efs-id>:/ /mnt/efs efs defaults,_netdev,tls 0 0" | sudo tee -a /etc/fstab

# 마운트 확인
df -h | grep efs
✅
검증: 두 EC2에서 /mnt/efs에 파일 생성 후, 다른 EC2에서 동일 파일이 보이는지 확인

Query from S3
S3에 데이터를 업로드하고 Athena로 SQL 쿼리하는 시나리오
아키텍처 구성
CSV/JSON 데이터
→
S3 버킷
→
Athena
→
SQL 결과
1
S3 버킷 생성 (데이터용 + 결과용)
bash · AWS CLI
# 데이터 저장 버킷 생성
aws s3 mb s3://<data-bucket-name> \
  --region ap-northeast-2

# Athena 쿼리 결과 버킷 생성
aws s3 mb s3://<athena-result-bucket> \
  --region ap-northeast-2
2
샘플 CSV 데이터 업로드
bash · AWS CLI
# 샘플 CSV 파일 생성
cat <<EOF > /tmp/sample.csv
id,name,age,city
1,Alice,30,Seoul
2,Bob,25,Busan
3,Charlie,35,Incheon
EOF

# S3에 업로드 (파티션 구조 권장)
aws s3 cp /tmp/sample.csv \
  s3://<data-bucket-name>/data/sample.csv
3
Athena 워크그룹 및 데이터베이스 설정
bash · AWS CLI
# Athena 워크그룹 생성 (결과 위치 지정)
aws athena create-work-group \
  --name my-workgroup \
  --configuration ResultConfiguration={OutputLocation=s3://<athena-result-bucket>/results/}

# 데이터베이스 생성 쿼리 실행
aws athena start-query-execution \
  --query-string "CREATE DATABASE IF NOT EXISTS mydb;" \
  --work-group my-workgroup
4
Athena 외부 테이블 생성
bash · AWS CLI
# CREATE TABLE DDL 파일 작성
cat <<'EOF' > /tmp/create_table.sql
CREATE EXTERNAL TABLE IF NOT EXISTS mydb.users (
  id     INT,
  name   STRING,
  age    INT,
  city   STRING
)
ROW FORMAT DELIMITED
  FIELDS TERMINATED BY ','
STORED AS TEXTFILE
LOCATION 's3://<data-bucket-name>/data/'
TBLPROPERTIES ('skip.header.line.count'='1');
EOF

# 테이블 생성 쿼리 실행
QUERY_ID=$(aws athena start-query-execution \
  --query-string file:///tmp/create_table.sql \
  --query-execution-context Database=mydb \
  --work-group my-workgroup \
  --query "QueryExecutionId" --output text)

echo "Query ID: $QUERY_ID"
5
S3 Select / Athena SQL 쿼리 실행
bash · AWS CLI
# Athena SQL 쿼리 실행
QUERY_ID=$(aws athena start-query-execution \
  --query-string "SELECT * FROM mydb.users WHERE age > 28;" \
  --query-execution-context Database=mydb \
  --work-group my-workgroup \
  --query "QueryExecutionId" --output text)

# 쿼리 완료 대기
aws athena get-query-execution \
  --query-execution-id $QUERY_ID \
  --query "QueryExecution.Status.State"

# 결과 조회
aws athena get-query-results \
  --query-execution-id $QUERY_ID \
  --query "ResultSet.Rows[*].Data[*].VarCharValue"
💡
S3 Select도 사용 가능: aws s3api select-object-content로 S3 객체를 직접 SQL로 쿼리할 수 있음 (Athena 없이)

Fine-grained IAM Policy
특정 리소스/조건에만 접근을 허용하는 세분화된 IAM 정책 생성 시나리오
⚠️
핵심 개념: Least Privilege (최소 권한 원칙) — 필요한 리소스, 액션만 정확히 지정
1
특정 S3 버킷만 접근 허용 정책
bash · AWS CLI
# 정책 JSON 작성 (특정 버킷 읽기 전용)
cat <<'EOF' > /tmp/s3-readonly-policy.json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowSpecificBucketRead",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::<bucket-name>",
        "arn:aws:s3:::<bucket-name>/*"
      ]
    }
  ]
}
EOF

# IAM 정책 생성
aws iam create-policy \
  --policy-name S3SpecificBucketReadOnly \
  --policy-document file:///tmp/s3-readonly-policy.json
2
특정 리전/태그 조건 기반 EC2 정책
bash · AWS CLI
# 태그 조건부 EC2 제어 정책
cat <<'EOF' > /tmp/ec2-tag-policy.json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowEC2WithTag",
      "Effect": "Allow",
      "Action": [
        "ec2:StartInstances",
        "ec2:StopInstances",
        "ec2:DescribeInstances"
      ],
      "Resource": "arn:aws:ec2:ap-northeast-2:<account-id>:instance/*",
      "Condition": {
        "StringEquals": {
          "ec2:ResourceTag/Environment": "dev"
        }
      }
    }
  ]
}
EOF

aws iam create-policy \
  --policy-name EC2DevEnvironmentOnly \
  --policy-document file:///tmp/ec2-tag-policy.json
3
IAM 사용자/역할 생성 및 정책 연결
bash · AWS CLI
# IAM 사용자 생성
aws iam create-user --user-name <username>

# 정책 ARN 확인
aws iam list-policies --scope Local \
  --query "Policies[?PolicyName=='S3SpecificBucketReadOnly'].Arn" \
  --output text

# 사용자에게 정책 연결
aws iam attach-user-policy \
  --user-name <username> \
  --policy-arn arn:aws:iam::<account-id>:policy/S3SpecificBucketReadOnly

# IAM 역할(Role)에 정책 연결하는 경우
aws iam create-role \
  --role-name MyAppRole \
  --assume-role-policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Principal":{"Service":"ec2.amazonaws.com"},"Action":"sts:AssumeRole"}]}'

aws iam attach-role-policy \
  --role-name MyAppRole \
  --policy-arn arn:aws:iam::<account-id>:policy/EC2DevEnvironmentOnly

MFA 조건 추가 (고급)
bash · AWS CLI
# MFA 인증된 사용자만 허용하는 조건 정책
cat <<'EOF' > /tmp/mfa-policy.json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyWithoutMFA",
      "Effect": "Deny",
      "NotAction": [
        "iam:CreateVirtualMFADevice",
        "iam:EnableMFADevice",
        "sts:GetSessionToken"
      ],
      "Resource": "*",
      "Condition": {
        "BoolIfExists": {
          "aws:MultiFactorAuthPresent": "false"
        }
      }
    }
  ]
}
EOF

aws iam create-policy \
  --policy-name RequireMFA \
  --policy-document file:///tmp/mfa-policy.json

MySQL with Lambda
RDS MySQL 생성 후 Lambda 함수와 VPC 내에서 연동하는 시나리오
아키텍처 구성
API / 이벤트
→
Lambda (VPC)
→
RDS MySQL
Lambda와 RDS가 동일 VPC/서브넷에 위치
1
RDS MySQL 서브넷 그룹 & 보안 그룹 생성
bash · AWS CLI
# RDS 서브넷 그룹 생성 (private 서브넷 2개 이상 필요)
aws rds create-db-subnet-group \
  --db-subnet-group-name mysql-subnet-group \
  --db-subnet-group-description "MySQL subnet group" \
  --subnet-ids <subnet-id-1> <subnet-id-2>

# RDS용 Security Group 생성
aws ec2 create-security-group \
  --group-name rds-sg \
  --description "RDS MySQL SG" \
  --vpc-id <vpc-id>

# Lambda SG에서 3306 허용
aws ec2 authorize-security-group-ingress \
  --group-id <rds-sg-id> \
  --protocol tcp \
  --port 3306 \
  --source-group <lambda-sg-id>
2
RDS MySQL 인스턴스 생성
bash · AWS CLI
# RDS MySQL 생성
aws rds create-db-instance \
  --db-instance-identifier my-mysql-db \
  --db-instance-class db.t3.micro \
  --engine mysql \
  --engine-version 8.0 \
  --master-username admin \
  --master-user-password <password> \
  --db-name mydb \
  --db-subnet-group-name mysql-subnet-group \
  --vpc-security-group-ids <rds-sg-id> \
  --no-publicly-accessible \
  --backup-retention-period 1 \
  --allocated-storage 20

# RDS 생성 완료 대기 (약 5~10분)
aws rds wait db-instance-available \
  --db-instance-identifier my-mysql-db

# 엔드포인트 주소 확인
aws rds describe-db-instances \
  --db-instance-identifier my-mysql-db \
  --query "DBInstances[0].Endpoint.Address" \
  --output text
3
Lambda 실행 역할 생성
bash · AWS CLI
# Lambda 신뢰 정책
cat <<'EOF' > /tmp/lambda-trust.json

  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",

4
Lambda 함수 코드 작성 및 패키징
python · lambda_function.py
# lambda_function.py
import json
import pymysql
import os

def lambda_handler(event, context):
    conn = pymysql.connect(
        host=os.environ['DB_HOST'],
        user=os.environ['DB_USER'],
        password=os.environ['DB_PASS'],
        database=os.environ['DB_NAME'],
        connect_timeout=5
    )
    
    with conn.cursor() as cursor:
        cursor.execute("SELECT VERSION()")
        result = cursor.fetchone()
    
    conn.close()
    
    return {
        'statusCode': 200,
        'body': json.dumps({'mysql_version': result[0]})
    }
bash · 패키징
# pymysql 설치 및 zip 패키징
mkdir -p /tmp/lambda-pkg
pip install pymysql -t /tmp/lambda-pkg/
cp lambda_function.py /tmp/lambda-pkg/
cd /tmp/lambda-pkg && zip -r /tmp/lambda.zip .
5
Lambda 함수 생성 (VPC 설정 포함)
bash · AWS CLI
# Lambda 함수 생성 (VPC + 환경변수)
aws lambda create-function \
  --function-name mysql-query-fn \
  --runtime python3.11 \
  --role arn:aws:iam::<account-id>:role/lambda-rds-role \
  --handler lambda_function.lambda_handler \
  --zip-file fileb:///tmp/lambda.zip \
  --timeout 30 \
  --vpc-config SubnetIds=<subnet-id-1>,<subnet-id-2>,SecurityGroupIds=<lambda-sg-id> \
  --environment Variables="{
    DB_HOST=<rds-endpoint>,
    DB_USER=admin,
    DB_PASS=<password>,
    DB_NAME=mydb
  }"

# Lambda 테스트 실행
aws lambda invoke \
  --function-name mysql-query-fn \
  --payload '{}' \
  /tmp/response.json && cat /tmp/response.json
✅
검증: response.json에서 mysql_version 값이 반환되면 연동 성공. CloudWatch Logs에서 실행 로그 확인 가능
